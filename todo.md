## 🎯 开发规范

### 提交信息规范 (Conventional Commits)

- `feat`: 新功能
- `fix`: 修复问题
- `docs`: 文档修改
- `style`: 代码格式修改（不影响功能）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

示例：
```
feat: add batch PDF printer component
fix: handle Chinese characters in printer names
docs: update README with installation instructions
```

### 文件命名规范

- 组件文件: `ComponentName.tsx`
- 样式文件: `ComponentName.module.css`
- 工具函数: `camelCase.ts`
- 常量定义: `CONSTANTS.ts`

### 组件开发规范

1. 每个工具一个独立目录：`src/components/ToolName/`
2. 包含 `index.ts` 导出主组件
3. 样式使用 CSS Modules
4. 复杂组件拆分子组件
5. 后端命令需要在 `lib.rs` 注册
